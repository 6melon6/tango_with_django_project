# Diet Tracker API 接口文档

## 基本信息

- **Base URL**: `http://localhost:8000`
- **认证方式**: Session Authentication (Django)
- **数据格式**: Form Data / JSON
- **字符编码**: UTF-8

---

## 1. 用户认证

### 1.1 显示注册表单

```
GET /register/
```

**描述**: 获取用户注册页面

**认证**: 不需要

**响应**: HTML 页面

---

### 1.2 用户注册

```
POST /register/
```

**描述**: 创建新用户账号

**认证**: 不需要

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| email | string | 是 | 电子邮箱 |
| password1 | string | 是 | 密码 |
| password2 | string | 是 | 确认密码 |

**成功响应**:
- 状态码: 302
- 跳转: `/dashboard/`

**失败响应**:
- 状态码: 200
- 返回: 包含错误信息的注册表单

---

### 1.3 显示登录表单

```
GET /login/
```

**描述**: 获取用户登录页面

**认证**: 不需要

**响应**: HTML 页面

---

### 1.4 用户登录

```
POST /login/
```

**描述**: 用户身份验证

**认证**: 不需要

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

**成功响应**:
- 状态码: 302
- 跳转: `/dashboard/` 或 `next` 参数指定页面

**失败响应**:
- 状态码: 200

**描述**: 结束用户会话

## 3. 饮食记录

### 3.1 获取记录列表

```
GET /records/
```

**描述**: 查看指定日期的饮食记录

**认证**: 需要

**查询参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| date | string | 否 | 日期 (YYYY-MM-DD)，默认今天 |

**响应数据**:

```json
{
  "entries": [...],
  "selected_date": "2026-02-26",
  "total_calories": 1850.5
}
```

---

### 3.2 显示添加记录表单

```
GET /record/add/
```

**描述**: 获取添加饮食记录的表单页面

**认证**: 需要

**响应**: HTML 表单页面

---

### 3.3 添加饮食记录

```
POST /record/add/
```

**描述**: 创建新的饮食记录

**认证**: 需要

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| food | integer | 是 | 食物 ID |
| quantity | decimal | 是 | 数量，必须 > 0，最多 6 位数字 1 位小数 |
| meal_type | string | 是 | 餐次: breakfast/lunch/dinner/snack |
| record_date | date | 是 | 记录日期 (YYYY-MM-DD)，不能是未来 |
| notes | string | 否 | 备注，最多 500 字符 |

**成功响应**:
- 状态码: 302
- 跳转: `/records/`

**失败响应**:
- 状态码: 200
- 返回: 包含错误信息的表单

---

**描述**: 获取编辑饮食记录的表单页面

## 4. 历史查询

### 4.1 查询历史记录

```
GET /history/
```

**描述**: 按日期或日期范围查询历史记录

**认证**: 需要

**查询参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| date | string | 否 | 单日查询 (YYYY-MM-DD) |
| start_date | string | 否 | 范围查询开始日期 |
| end_date | string | 否 | 范围查询结束日期 |
| meal_type | string | 否 | 餐次筛选: breakfast/lunch/dinner/snack |

**响应数据**:

```json
{
  "entries": [...],
  "chart_data": [
    {
      "record_date": "2026-02-25",
      "total": 1850.5
    },
    {
      "record_date": "2026-02-26",
      "total": 1920.0
    }
  ],
  "has_results": true
}
```

## 数据模型

### User (用户)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer | 主键 |
| username | string | 用户名 |
| email | string | 邮箱 |
| password | string | 密码（哈希） |

---

### FoodCategory (食物分类)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer | 主键 |
| category_name | string(100) | 分类名称（唯一） |
| description | text | 描述 |

---

### Food (食物)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer | 主键 |
| food_name | string(100) | 食物名称 |
| category_id | integer | 分类 ID（外键） |
| calories_per_100g | integer | 每 100 克卡路里 |
| unit | string(20) | 单位 |

---

### FoodRecord (饮食记录)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer | 主键 |
| user_id | integer | 用户 ID（外键） |
| food_id | integer | 食物 ID（外键） |
| quantity | decimal(6,1) | 数量 |
| meal_type | string | 餐次: breakfast/lunch/dinner/snack |
| record_date | date | 记录日期 |
| notes | text | 备注 |

**计算属性**:
- `total_calories = (food.calories_per_100g × quantity) / 100`

---

## 状态码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 成功（返回页面或数据） |
| 302 | 重定向 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

---

## 错误响应格式

表单验证错误会在页面中显示，格式为：

```html
<div class="alert alert-danger">
  <ul>
    <li>字段名: 错误信息</li>
  </ul>
</div>
```
